# Installation

## Docker (recommended)

The easiest way to run RMS is with Docker. The image is published as `pixelotes/rms:arm64`.

```bash
docker run -d \
  --name rms \
  -p 8082:8082 \
  -v ./config:/app/config \
  -v /media:/app/media:ro \
  --memory=128m \
  pixelotes/rms:arm64
```

The `:ro` flag mounts media as read-only. RMS never modifies your media files during normal operation (only crawlers write NFO/artwork files).

## Docker Compose

```yaml
services:
  rms:
    image: pixelotes/rms:arm64
    container_name: raspberry-media-server
    ports:
      - "8082:8082"
    volumes:
      - ./config:/app/config
      - /media:/app/media:ro
    deploy:
      resources:
        limits:
          memory: 128M
    restart: unless-stopped
```

## From Source

Requires Go 1.24+ and FFmpeg.

```bash
git clone https://github.com/pixelotes/rms.git
cd rms

# Build all binaries
CGO_ENABLED=0 go build -o rms ./cmd/rms
CGO_ENABLED=0 go build -o metacrawler ./cmd/metacrawler
CGO_ENABLED=0 go build -o subcrawler ./cmd/subcrawler

# Run
./rms -config config/config.yml
```

## Binaries

RMS ships three binaries:

| Binary | Purpose | Runs as |
|--------|---------|---------|
| `rms` | Media server | Long-running daemon |
| `metacrawler` | Metadata & artwork downloader | On-demand or scheduled |
| `subcrawler` | Subtitle downloader | On-demand or scheduled |

The crawlers are separate binaries by design. They don't consume server memory when idle, and can be run independently from the command line.

## Requirements

- **FFmpeg** — required for video duration probing, remux streaming, and thumbnail generation
- **Raspberry Pi** — tested on Pi 4/5 with 1GB+ RAM; RMS itself uses ~9 MB
- **Media files** — organized in Kodi/Jellyfin standard directory structure

## Directory Structure

RMS expects media organized as:

```
/media/
  Movies/
    Movie Name (Year)/
      Movie Name (Year).mp4
      movie.nfo          # optional, auto-generated by metacrawler
      poster.jpg         # optional, auto-downloaded
      fanart.jpg         # optional, auto-downloaded
  TV/
    Show Name (Year)/
      tvshow.nfo
      poster.jpg
      fanart.jpg
      Season 1/
        Show.S01E01.mp4
        Show.S01E01.nfo
        Show.S01E02.mp4
      Season 2/
        ...
  Anime/
    Anime Name (Year)/
      S01/
        [Group] Anime Name - 001 - Episode Title.mkv
```

This is the same structure used by Kodi, Jellyfin, Emby, and Plex.
